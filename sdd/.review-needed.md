## 2026-04-28 — Test drift on REQ-PWA-003 AC 4 (HIGH)

**Trigger:** push 8418029 on `develop` — refactor of brand-tap from document capture-phase listeners to element-bound listeners in `src/scripts/page-effects.ts` and `public/scripts/page-effects.js`.

**Spec impact:** none. REQ-PWA-003 AC 4 describes user-observable behavior (scroll-to-top on unfiltered /digest, fall-through on filtered /digest, never-intercept on modifier-clicks). The refactor preserves all four observable contracts. The acceptance criteria text remains accurate; no edit to `sdd/pwa.md` is required.

**Test impact:** `tests/pwa/mobile-layout.test.ts` contains two AC-4 tests written against the previous implementation strategy rather than the AC contract. The refactor invalidates three regex assertions:

1. Line 125 — `expect(effectsSource).toMatch(/closest[\s\S]{0,80}a\[data-brand-home\]/);` no longer matches; the new code uses `document.querySelector<HTMLAnchorElement>('a[data-brand-home]')` at bind time, not `closest()` on the event target.
2. Lines 144-146 — `addEventListener('click', ..., true)` capture-phase assertion no longer matches; listeners are now bound directly to the brand element without the capture flag.
3. Line 136 test name — "runs in capture phase + stopPropagation so it wins the race with Astro ClientRouter" is now misleading; the new strategy relies on element-target-first dispatch ordering plus `stopPropagation`, not the capture flag.

**Recommended action:** rewrite the two AC-4 tests as behavioral tests rather than source-pattern tests. The tests should assert the four observable contracts of AC 4 directly:

- Contract 1: brand `<a>` `href` resolves to `/digest` for authenticated users.
- Contract 2: simulated click on the brand while `location.pathname === '/digest'` and `location.search === ''` calls `window.scrollTo({ top: 0, ... })` and does not navigate.
- Contract 3: simulated click on the brand while `location.search !== ''` lets the default navigation proceed (no `preventDefault`).
- Contract 4: simulated click with `metaKey`/`ctrlKey`/`shiftKey`/`altKey` set, or `button !== 0`, never calls `preventDefault` regardless of pathname.

Behavioral tests survive future implementation refactors (capture-phase vs element-bound vs delegated, `closest()` vs `querySelector`, etc.) because they bind to the AC, not the code.

**Lane:** test repair is the developer's lane (or `tdd-guide` agent), not spec-reviewer's. Flagged here because CI will fail on this push for the reasons above and the user should know the refactor itself is spec-conformant.
