## 2026-04-27 — REQ-AUTH-002 AC 5 (explicit-refresh endpoint) coverage gap

**Severity:** HIGH (sub-case gap on a freshly-added AC; not auto-fixable by spec-reviewer).

**What's missing:** REQ-AUTH-002 AC 5 documents the explicit `/api/auth/refresh` endpoint contract that was added in this push to address two HIGH code-review findings — (a) it must always force-rotate when the refresh cookie is valid regardless of remaining access-JWT lifetime, (b) on every failure path both cookies are cleared on the response. The endpoint source at `src/pages/api/auth/refresh.ts` carries `Implements REQ-AUTH-002, REQ-AUTH-008` and the code is correct, but no test file imports or exercises that handler. All existing tests in `tests/auth/refresh-tokens.test.ts` go through the middleware `loadSession` path, not the explicit endpoint POST handler.

**Why it isn't auto-demoted:** REQ-AUTH-002 already has dedicated tests for ACs 1–4 (middleware path, JWT shape, session_version, OAuth completion). The Phase-2 5c count gate passes at the REQ level (test_count ≥ ac_count). The under-pinned AC is a sub-case the count heuristic does not catch — same pattern as the AC 3 entry above.

**Why it matters:** The push specifically fixed two correctness bugs in this endpoint (the 401 path collapsed Set-Cookie to one entry, and the success path silently no-op'd when the access JWT was still valid). Without tests, regressions on either fix would only surface in production.

**Recommended fix (next push):** add `tests/auth/refresh-endpoint.test.ts` (or extend `tests/auth/refresh-tokens.test.ts`) with at least:
1. Force-rotate semantics — call POST `/api/auth/refresh` with a valid refresh cookie and a still-live access JWT; assert response carries new access cookie + new refresh cookie (not the same value as input), and the old refresh row is now revoked.
2. Cookie-clearing on 401 — call with an invalid or absent refresh cookie; assert response status is 401 AND both `__Host-…` access and refresh cookies are cleared via two distinct `Set-Cookie` headers (not collapsed to one).
3. Grace-window collision behaviour — pre-revoke the refresh row inside the 30-second window with a surviving rotated child; assert response returns 200 with a fresh access JWT only (no new refresh cookie) and `rotated: false`.
4. Outside-window replay — pre-revoke the refresh row outside the 30-second window; assert reuse-detection fires, all rows for the user are revoked, response is 401 with both cookies cleared.
5. Origin-check — call with a foreign Origin header; assert 403 forbidden_origin.

**Lane:** `tdd-guide`'s lane, not spec-reviewer's. Logged here so the gap is visible until tested.
